  • 사용자 입력 자연어 처리를 위해 KoBERT, KoELECTRA 등 한국어에 특화된 경량화 NLP 모델을 활용하여 의도 및 핵심 키워드를 분석
  • 사용자 프로파일과 분석된 입력 데이터를 결합하여 개인화된 콘텐츠 추천 로직 구현(예: 협업 필터링, 콘텐츠 기반 필터링)
[DER-02] 큐레이션 기능 구현 로직(AI프레임워크에 맞춘 콘텐츠 추천 기능) [DER-02] AI 역량 프레임워크 기반 큐레이션 엔진 개발
  • 제시된 AI 역량 프레임워크(이해, 윤리, 활용성 등)를 기준으로 콘텐츠를 메타데이터화
  • 사용자의 학습 이력, 관심사 프로파일, 퀴즈 결과 등을 분석하여 AI 역량 수준을 진단하고, 부족한 역량을 보충할 수 있는 맞춤형 콘텐츠를 추천하는 규칙 기반/ML 기반 하이브리드 추천 엔진 설계
[DER-02] 토큰 사용량, 요청 로그, 예외 처리 등 API 호출 관리 체계 관리

  • 사용자별/서비스별 API 호출량을 실시간으로 추적하고, 임계치 도달 시 알림 및 자동 차단 기능을 구현하여 예산 관리
  • 모든 예외 상황에 대한 상세 로그를 기록하고, 에러 모니터링 등 대응 체계 마련
  • 외부 API 연동을 위한 인터페이스를 표준화하여 다양한 AI 모델(OpenAI, Anthropic, Google 등)을 유연하게 교체 및 확장할 수 있는 아키텍처
[DER-03] 주기별 구독 방식의 실생활 적용 콘텐츠 기획 [DER-03] 구독형 콘텐츠 제공 시스템 설계
  • 사용자가 관심 주제(프롬프트, AI 활용팁 등)를 선택하면, 설정된 주기(일별/주별/월별)에 맞춰 신규 콘텐츠를 인앱(혹은 홈페이지) 알림으로 제시하는 시스템 구축

DER-01: 콘텐츠 집필진 구성 및 원고 집필


>> 산출물: 콘텐츠별 집필진 계획안(리스트 포함) 및 콘텐츠별 원고 작성 기획안
세부 내용 해결 방안
전문가 기반 집필진 구성
  • AI 기술 전문가, AI 교육 전문가 등으로 구성
 전문가 기반 집필진 구성
  • AI 분야별(생성형 AI, AI 윤리, 교육공학) 최고 수준의 기술 전문가와 현장 경험이 풍부한 교육 전문가로 구성된 '드림팀' 구성
  • 각 전문가의 전문 분야와 역할을 명확히 정의한 프로필과 운영 계획서를 제출하여 전문성 증명
정책·기술 동향 기반 내용 구성
  • 국내외 AI윤리, 활용사례, AI역량 정책, 최신 기술 트렌드 반영한 내용 구성
  • 다양한 학습자 이해 수준을 고려한 추가자료, 시각화 등 콘텐츠 유형 최적화한 원고 집필
 정책·기술 동향 기반 내용 구성
  • 최신 AI 기술 동향(논문, 기술 블로그), 정책 보고서를 주간 단위로 분석하여 콘텐츠에 즉각 반영하는 'Agile 콘텐츠 개발 프로세스' 등 도입
  • 복잡한 개념을 쉽게 전달하기 위해 인포그래픽, 비유, 실제 사례 중심의 스토리텔링 기법을 적용하고, 학습자 수준(기초/심화)에 따른 맞춤형 자료 별도로 제공
집필진 구성에 대해서는 NIA가 주도적으로 진행하며 그 인원수와 범위는 NIA와 협의 후 확정
  • 그분야 전문가 혹은 석박사급의 연구원, 교수 등
 NIA 주도 집필진 구성 협력
  • NIA와의 긴밀한 협력을 위해 전담 커뮤니케이션 채널을 구축하고, 정기적인 협의를 통해 최적의 집필진 구성
  • 제안 단계에서 확보한 최고 수준의 전문가 풀(Pool)을 기반으로 NIA의 요구사항에 맞는 맞춤형 후보군 리스트 제시
제안서에는 즉시 투입가능한 집필전문가 후보군을 제시하고 전문가별 소속과 AI 주제별 전문 관련성을 제안 하여야 함
  • 원고 집필자, 내용 검토, 교수 설계자와 협업 추진
  • 집필 전문가 후보군은 '추진단' 관련 내용 참고
  • 콘텐츠 제작 기간 확보를 위해 품질을 유지하면서 집필 기간을 단축할 수 있는 방안 필수 제안
 즉시 투입 가능한 전문가 후보군 및 협업 방안 제시
  • 계약 즉시 프로젝트에 착수할 수 있는 핵심 집필진, 검토위원, 교수 설계 전문가 후보군의 상세 프로필 및 참여의향서 제출
  • '콘텐츠 모듈화' 전략을 통해 주제별 집필을 병렬로 진행하고, 표준화된 템플릿과 스타일 가이드를 제공하여 원고의 통일성을 확보함으로써 집필 기간을 30% 이상 단축하는 방안 제안. 협업툴(Notion, Slack) 등 활용.
집필진은 계약완료후 10일이내에 즉각 투입이 가능한 리스트를 2안 이상 제시하여야 하며 착수 후 2주안에 일별 콘텐츠 보급, 3주 안에 주별 콘텐츠 보급, 1개월이내 유료 API연계 콘텐츠 시범 적용 및 2개월안에 2종이상 체험형 콘텐츠 초고입고 신속한 집필진 투입 및 조기 성과물 제출 계획
  • 계약 후 10일 내 투입 가능한 전문가 리스트(주 집필진/예비 집필진) 2개 안을 제출하고, 즉시 착수 회의 진행
  • 마일스톤 기반의 구체적인 실행 계획 제시
    • 2주차: 일별 'AI 활용 팁' 카드뉴스 콘텐츠 보급 시작
    • 3주차: 주별 '생성형 AI 심화' 뉴스레터 보급 시작
    • 1개월차: 유료 API 연동 'AI 챗봇' 프로토타입 시범 운영
    • 2개월차: 'AI를 이겨라(퀴즈)', 'AI 윤리 딜레마' 체험형 콘텐츠 초안 제출

DER-02: AI실습·체험 통합 콘텐츠 기획 및 개발


>> 산출물: 개발 계획안, AI실습·체험 통합 콘텐츠(큐레이션 포함)
세부 내용 해결 방안
HTML5 기반 웹표준 콘텐츠 구현
  • 주요 브라우저 간 크로스 브라우징 지원 및 PC, 태블릿, 모바일 환경에서 호환 가능한 반응형 UI 설계
  • 디지털배움터 홈페이지 배너 탑재 등 탑재-운영 전반적인 기술 지원
 웹 표준 및 반응형 UI/UX 구축
  • 모든 콘텐츠는 HTML5, CSS3, JavaScript(ES6+) 표준을 준수하여 개발하며, 주요 브라우저(Chrome, Edge, Safari, Firefox)의 지원
  • 디지털배움터 기술팀과 협력하여 배너 및 콘텐츠 탑재를 위한 기술 가이드를 사전에 마련하고, 원활한 연동 보장
사용자 입력(텍스트, 선택값 등)과 프로파일(연령, 관심사 등)을 기반으로 NLP 분석 수행 NLP 기반 사용자 분석 엔진 적용
  • 사용자의 비정형 텍스트 입력을 분석하여 핵심 관심사와 역량 수준을 파악하는 경량 NLP 모듈 탑재
  • 분석된 결과를 바탕으로 사용자 프로필을 동적으로 업데이트하여, 시간이 지날수록 더욱 정교한 맞춤형 콘텐츠 추천이 가능하도록 설계
토큰 사용량, 요청 로그, 예외 처리 등 API 호출 관리 체계 관리 API 통합 관리 대시보드 구축
  • 외부 AI API(유료) 호출에 대한 실시간 모니터링 대시보드 구축
  • 이를 통해 일/주/월별 토큰 사용량, 비용, API 응답 시간, 오류 발생률을 시각적으로 추적하고, 예산 초과 방지를 위한 알림 시스템 운영
KWCAG 2.1 수준 AA 이상의 접근성 지침 반영 필요 웹 접근성 AA등급 준수
  • 설계 단계부터 웹 접근성 전문가가 참여하여 모든 UI/UX가 KWCAG 2.1 AA 등급 기준 충족 (예: 대체 텍스트, 키보드 인터페이스, 명도 대비 등)
  • 주기적인 자동화된 접근성 테스트와 스크린 리더 사용성 테스트로 접근성 유지
큐레이션 기능 구현 로직(AI프레임워크에 맞춘 콘텐츠 추천 기능)
  • 디지털배움터 온·오프라인 강좌, 콘텐츠 추천서비스 확대가 가능하도록 설계 및 기능 구현
 확장 가능한 큐레이션 시스템 설계
  • AI 역량 프레임워크를 기반으로 각 콘텐츠에 메타데이터 태그를 부여하고, 사용자의 활동 이력과 프로필을 분석하여 개인별 맞춤 콘텐츠 추천 엔진 개발
  • 향후 디지털배움터의 다른 온·오프라인 강좌 정보와도 연동하여 통합 추천이 가능하도록 확장성을 고려한 API 기반 설계
디지털배움터 홈페이지 및 오프라인 디지털배움터의 다양한 디지털 기기(PC,태블릿,모바일,키오스크 등) 탑재 시 고려 필수 멀티 디바이스 호환성 확보
  • 웹 기반 기술을 사용하여 PC, 태블릿, 모바일은 물론, 키오스크의 터치 인터페이스에도 최적화된 UI/UX 제공
  • 저사양 기기에서도 원활하게 구동될 수 있도록 경량화/최적화
개인정보 보호법, 정보통신망법 등 관련 법령 준수
  • 사용자의 프롬프트 입력에 개인정보가 포함되어 있을 경우 대응 조치 마련 필수
 강화된 개인정보보호 조치 마련
  • 프롬프트 제출 전, 개인정보(주민번호, 주소, 전화번호 등) 포함 여부를 감지하는 필터링 로직을 적용하고, 감지 시 사용자에게 경고 및 비식별화 조치 안내
  • 모든 데이터는 수집-저장-활용-파기 전 과정에 걸쳐 관련 법령 준수 및 관리
12.1~31 시범운영 추진(필수)
  • 시범운영에 필요한 유료토큰비용은 사업비 내에서 지출(예산 10백만원)
  • 하루 1,000명*30일*1명당 30회질문 이상으로 확보 필수
  • 유료 API사용량 증가 가능성에 대비한 예산 관리 방안 필요하며 API업데이트, 기능개선, 버그대응 등을 위한 유연한 유지보수 및 관리계획 수립 필요
 체계적인 시범운영 및 유지보수 계획 수립
  • 시범운영 기간(12/1~31) 동안 안정적인 서비스 제공을 위해 API 사용량 예측 모델을 기반으로 1000만원 예산 내에서 최대의 사용자 경험을 제공할 수 있도록 토큰 사용량 최적화
  • 예산 소진 방지를 위해 사용자별 일일 사용량 제한(Rate Limiting) 기능을 도입하고, 실시간 비용 모니터링을 통해 위험 관리
  • 시범운영 중 발견되는 버그 및 개선사항은 이슈 트래킹 시스템에 기록하고, 우선순위에 따라 신속하게 대응하는 유지보수 프로세스 운영

DER-03: 프레임워크 기반 AI 기초역량 콘텐츠 개발


>> 산출물: 모범 프롬프트 40종 + AI리터러시 교육 콘텐츠 5종
세부 내용 해결 방안
AI프레임워크 기반으로 각 영역별 주제, 콘텐츠 유형 기획
  • 각 영역별(이해 윤리,활용성, 지향성, 위험대처) 개발·보급이 시급한 주제 5개 이상을 제시하고 선정·보급확산 기획
